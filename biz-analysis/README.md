# Biz Analysis Skill — 深度业务分析框架

一个结构化、可复用的业务分析 Skill，帮助团队产出**有数据支撑、有竞品对标、有可执行优先级**的分析报告。

## 🎯 适用场景

| 场景 | 说明 |
|------|------|
| **商业化活动分析** | 重点放在效果数据 + 机制对标 |
| **用户增长分析** | 重点放在漏斗数据 + 用户分层 |
| **产品功能评估** | 重点放在使用数据 + 竞品对比 |
| **运营策略复盘** | 重点放在时间线全景 + 根因分析 |
| **新业务调研** | 重点放在竞品深度 + 行业规律 |

## 📦 物料清单

- `SKILL.md` — 核心 Skill 文档（完整分析框架）
- `TEMPLATE.md` — 快速应用模板（直接套用）
- `EXAMPLES/` — 成功案例参考
- `CHECKLIST.md` — 输出核验清单

## 🚀 快速开始

### 1. 复制 Skill 文件

将 `SKILL.md` 复制到你的 Obsidian vault 的 `.claude/skills/` 目录下：

```bash
cp SKILL.md ~/Documents/Obsidian\ Vault/.claude/skills/biz-analysis/
```

### 2. 在 Obsidian 中调用

在任意笔记中，使用 @ 引用：

```
@.claude/skills/biz-analysis/SKILL.md 帮我分析一下 [业务名称] 的 [核心问题]
```

### 3. 按模板提供信息

为了让 AI 给出精准分析，请按模板提供背景、目标、约束等信息（见 `TEMPLATE.md`）。

## 📋 核心框架

### 五步分析法

```
第一步：收集输入（必须有）
  ├─ 历史效果数据（核心指标实际数据）
  ├─ 产品/业务背景（模式、KPI、用户结构）
  └─ 问题或目标描述（当前瓶颈）

第二步：构建分析 Prompt
  ├─ 【背景】（产品+变现+用户+KPI数值）
  ├─ 【目标】（按优先级排列）
  └─ 【约束/已知信息】

第三步：标准分析结构（6个模块）
  1. 全景梳理（时间线/事件全览）
  2. 效果对比（好的 vs 差的，各3个，有根因）
  3. 规律提炼（铁律/可复用规则）
  4. 竞品对标（至少3个竞品，聚焦"我们能学什么"）
  5. ��户分层（不同用户群的差异化策略）
  6. 可执行优先级（本周/2-4周/1-3个月）

第四步：迭代循环（质疑清单）
  ├─ "这个结论怎么从数据推导的？"
  ├─ "这个建议被竞品验证过吗？"
  ├─ "最差案例的根因找到了吗？"
  └─ "这个结论在实际情况下能不能用？"

第五步：输出核验（5个检查点）
  ├─ 每个结论有数据支撑
  ├─ 有实际数字
  ├─ 优先级可直接开工
  ├─ 识别分析局限性
  └─ 竞品数据有来源
```

## 💡 核心原则

### 1. 数据驱动
- 每个结论必须有数据支撑（内部数据 or 竞品数据）
- 用数字说话，不用形容词
- 百分比要搭配绝对值（如"转化率提升20%" → "从5%提升到6%"）

### 2. 根因分析
- 不只说结果，要说"为什么好""为什么差"的根因
- 差案例的根因分析比成功案例更有价值

### 3. 可执行性
- 优先级清单必须是"可直接开工"的
- 不是"应该做AB测试"，而是"用X和Y做AB测试，时间窗口Z，主指标W"

### 4. 竞品对标
- 至少3个竞品的实际数据
- 聚焦"我们能学什么"，而不是"他们在做什么"

## 📖 使用示例

### 示例1：天天练活动策略分析

```markdown
@.claude/skills/biz-analysis/SKILL.md 帮我分析天天练活动策略

背景：
- 产品：天天练（K12学习产品）
- 核心变现：活动付费（付费率约4%）
- 核心变量：皮肤（用户喜好直接决定付费）
- 当前卡点：活动参与率连续2周下降10%

目标：
1. 找到活动参与率下降的根本原因
2. 提出可落地的提升方案（3个月内提升20%）
3. 沉淀可复用的活动设计checklist

材料：
- 已提供：活动数据看板截图、用户反馈文档

约束：
- 活动预算有限，不能大规模增加奖励
- 需保持现有活动节奏（每周1-2个活动）
```

**输出**：基于6份文档，2天产出的深度分析报告（见 `EXAMPLES/`）。

### 示例2：新业务调研

```markdown
@.claude/skills/biz-analysis/SKILL.md 帮我分析UGC地图生态

背景：
- 产品：天天练UGC地图
- 核心KPI：人均地图时长（当前180s）
- 问题：新图进入热图的转化率仅15%

目标：
1. 优化新图发现机制，提升转化率至30%
2. 沉淀创作者激励SOP
```

## 🔄 迭代流程

```
第一稿 → 质疑清单（问AI） → 补充数据 → 第二稿 → 团队评审 → 定稿
```

**关键**：第一稿出来后，**不要直接用**。按质疑清单逐条问 AI，补充数据，迭代至少一轮。
